# Scalp Fill Rate Improvement Recommendations
*Generated: 2026-04-01 17:18*

## Problem
Images #2 and #3 (unfilled scalp) share a common pattern: BTC made a large directional move in the **first 60–90 seconds** of the window. By the time the scalp strategy activated at T-150, market makers had already repriced the ask above `max_price`, causing the IOC to silently fail.

Image #1 (filled) worked because the move was gradual enough that the ask hadn't fully repriced by the scalp entry.

---

## Recommendations (Priority Order)

### 1. Move Scalp Entry Earlier — T-200 to T-240 (High Impact, Low Effort)
- Current: `entry_window_seconds = 150`
- Change to: 200–240 seconds
- Fires before market makers finish repricing after a fast early move
- The probability model (`_estimate_prob_from_delta`) already handles lower confidence at earlier entries via `remaining_vol` scaling — a large sustained delta still produces a meaningful z-score

### 2. GTC Maker for Large Early Moves (High Impact, Medium Effort)
- When a large delta (>0.30%) forms early in the window, the ask is already expensive — an IOC taker at a capped price will never fill
- Instead, post a GTC limit bid slightly below the current ask (e.g., ask − 0.01)
- Rests on the book; fills if a counterparty hits it before window close
- Cancelled at window close if unfilled (existing cleanup sweep handles this)
- Apply when: large delta detected at T-200 or earlier

### 3. Dynamic max_price from Actual Ask (Medium Impact, Low Effort)
- Current: `max_price = prob_win - 0.05 + buffer` — fixed formula, ignores the live order book
- `get_ask_depth()` is already called but used only as a binary liquidity check
- Change: if `best_ask ≤ prob_win` (positive EV exists), set `max_price = min(best_ask, prob_win - min_edge/2)`
- Accepts thinner edge rather than zero fill — still profitable expectation

### 4. Sustained-Drift Detection Mode (Medium Impact, Medium Effort)
- Images #2 and #3 signature: `|delta_pct| > 0.20%` AND `vol_5s / vol_60s < 1.5` (steady move, not a spike)
- Opposite of the Fade strategy's spike filter
- Add a "sustained drift" trigger inside the scalp that bypasses the T-150 gate and fires at T-200+ for this pattern only
- Regular scalp (T-150 to T-10) continues to handle noisier, later-forming moves
